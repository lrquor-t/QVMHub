2. 快速入门

基础设施配置

创建网络

QVMConsole 采用 OVS（Open vSwitch）交换机体系，提供两种网络模式：

NAT 模式（公有云推荐）

虚拟机不需要从外部直接访问的场景。系统内部 DHCP 服务统一管理网络和分配 IP，若需从外部访问虚拟机，可使用端口转发功能。

桥接模式（私有云推荐）

将物理网口直接与虚拟机打通，由上级路由器统一管理。步骤：

  1. 创建物理网桥：选择物理网卡创建 OVS 网桥
  2. 创建交换机：绑定 OVS 网桥，配置 VLAN ID

创建存储池

每个卡片对应一个物理磁盘。支持两种方式：

  • 物理磁盘作为存储（新手推荐）：在界面中根据磁盘容量进行分区
  • 多物理磁盘合并为 LVM 卷：适用于大容量存储灵活管理

配置 ISO 存放目录（可选）

开通"我的存储"功能后在系统设置中修改 ISO 存储位置。

使用模板创建虚拟机

下载预置模板

通过以下链接下载预置模板压缩包：

  • 123网盘：https://1852139552.share.123865.com/123pan/OAgIvd-fYoOH
  • 夸克网盘：https://pan.quark.cn/s/a5c1097063cd
  • 百度网盘：https://pan.baidu.com/s/1iEftpZkQgoCWTCS5PSSlXw?pwd=6f9y

导入模板

通过网页上传或 SFTP 上传到设备中，然后在模板管理页面完成导入。

从模板创建虚拟机

在新虚拟机表单中选择「从模板快速克隆」即可使用模板创建虚拟机。

链式克隆警告：若选择链式克隆方式创建虚拟机，必须确保模板不能丢失或损坏，否则将造成克隆后的虚拟磁盘损坏！

3. 进阶内容

制作模板

工作原理

模板制作本质上是将虚拟机的磁盘文件进行稀疏复制（cp --sparse=always），支持两种克隆模式：链式克隆（速度快、占用小）和完整克隆（完全独立）。

制作 Windows 模板

  1. 下载驱动和工具（virtio 驱动 ISO、CloudbaseInit）
  2. 准备系统 ISO 文件
  3. 创建虚拟机（选择系统 ISO + virtio 驱动 ISO）
  4. 进入审计模式（Ctrl + Shift + F3）
  5. 安装驱动（virtio-win-gt-x64.msi）
  6. 安装软件和环境
  7. 安装 CloudbaseInit（最后两个选项不要勾选）
  8. 执行 Sysprep 封装
  9. 制作模板

制作 Linux 模板

# Debian/Ubuntu
apt-get install -y cloud-init cloud-guest-utils

# CentOS/RHEL
yum install -y cloud-init cloud-utils-growpart

# Fedora/Rocky/Alma
dnf install -y cloud-init cloud-utils-growpart

然后直接关机，制作模板即可。

制作 OpenWrt / FnOS 模板

OpenWrt：从固件官网下载 img 导入虚拟机确保正常开机，然后直接关机制作模板。

FnOS：前往飞牛官网下载 ISO 正常安装飞牛后进入系统，不需要初始化飞牛，直接关机制作模板。

Linux 启动后命令

允许在系统完成初始化后后台自动执行命令，可用于配置业务环境或更换镜像源。

bash <(curl -sSL https://gitee.com/SuperManito/LinuxMirrors/raw/main/ChangeMirrors.sh) \
  --source mirrors.aliyun.com --protocol http \
  --install-epel true --backup false --upgrade-software false \
  --clean-cache false --ignore-backup-tips

安装 Windows

进入安装流程

在出现「Press any key to boot from CD or DVD」提示时，需立即按下键盘任意键进入安装流程。

加载 VirtIO 磁盘驱动

在「选择安装 Windows 的位置」步骤中，点击加载驱动，选择 CD 驱动器中的 amd64/win11 目录（Server 2022 对应 2k22）。

加载 VirtIO 网络驱动

Windows 11 需要在网络配置阶段安装网络驱动。

补全 VirtIO 驱动 & 安装 QEMU Guest Agent

进入桌面后运行 virtio-win-gt-x64.msi，然后安装 Guest Agent 程序。

安装 Linux

安装 QEMU Guest Agent

# Debian/Ubuntu
sudo apt update && sudo apt install -y qemu-guest-agent
sudo systemctl start qemu-guest-agent
sudo systemctl enable qemu-guest-agent

# RHEL/CentOS/Fedora/Rocky
sudo dnf install -y qemu-guest-agent
sudo systemctl start qemu-guest-agent
sudo systemctl enable qemu-guest-agent

4. 常见问题

面板问题

升级后存在很多问题以及页面不匹配：大部分是浏览器缓存造成的。

  • 方法一：F12 → 网络选项卡 → 勾选"停用缓存" → 刷新
  • 方法二：F12 → 应用选项卡 → "清除站点数据" → 刷新

已知报错

无法载入 AppArmor 侧写：passt 包与 AppArmor 不兼容，请更新软件包：

sudo apt update && sudo apt upgrade

获取域 xxx 元数据失败：中文系统造成命令输出差异，将系统语言改为英语后重启：

sudo dpkg-reconfigure locales
# 选择 en_US.UTF-8
sudo reboot

系统初始化原理

Linux 初始化（cloud-init NoCloud）

  • 身份重置：清理 machine-id、DHCP 租约
  • cloud-init 配置清理：删除安装器遗留配置
  • NoCloud Seed 注入：meta-data + user-data
  • 密码处理：virt-customize --password
  • 用户名重命名：离线 usermod

Windows 初始化（CloudbaseInit + ConfigDrive）

  • 使用 CloudbaseInit 配合 ConfigDrive 方式
  • 生成 Unattend.xml 跳过 OOBE 向导
  • 创建 Config Drive ISO 挂载到虚拟机
  • 通过 QEMU Guest Agent 自动弹出 CD-ROM

FnOS 初始化

通过 virt-customize 直接修改系统文件，不依赖 cloud-init。支持三种设备 ID 策略：自动重置、保留原始 ID、自定义 ID。

模板管理原理

模板元数据

每个模板由 .qcow2 磁盘文件和 .meta.json 元数据文件组成。字段：type、category、boot_type、template_user、cloud_init_mode、default_config、template_uid、node_id、parent_node_id 等。文件会被自动设置为不可变属性（chattr +i）。

树形结构

采用树形链式结构管理版本关系。从模板 A 克隆虚拟机再制作新模板 B 时，B 自动成为 A 的子节点。

克隆模式

模式原理优点缺点
链式克隆基于模板创建差分磁盘（backing file）毫秒级创建，节省存储依赖模板存在
完整克隆完整复制模板数据独立运行创建较慢，占用完整空间
原生链式克隆直接使用模板不修改配置最快速度仅管理员可用

模板删除策略

  • 级联删除（cascade）：删除模板及其整棵子树
  • 提升子节点（promote_children）：通过 qemu-img rebase 指向父模板
  • 热提升子节点（promote_children_hot）：支持运行中 VM 热切换

2. 虚拟机

创建方式

  • ISO 镜像安装：全新部署，完全控制安装过程，多 ISO 支持
  • 模板快速克隆：批量部署，秒级创建，支持链式/完整克隆
  • 原生链式克隆：快速测试，不修改模板配置
  • 导入已有磁盘：支持 qcow2、raw、vmdk、vhd/vhdx、img 格式自动转换

基础配置

  • 命名规则：字母、数字、短横线，最大 63 字符，唯一
  • 智能命名：随机生成 / 批量前缀+数字后缀
  • 应用场景：Web 应用、数据库、AI/机器学习、文件存储（智能推荐）
  • 系统类型：Linux、Windows、macOS、Other
  • 批量创建：自动计算总资源需求，检查配额

硬件规格

  • CPU 核心：1-32 核
  • CPU 限制：基于 Linux CFS 带宽控制
  • CPU 拓扑：自动/宿主默认/自定义（插槽+核心）
  • CPU 亲和性：绑定到特定物理核心
  • 动态内存：气球调度（virtio-balloon）/ 弹性内存（virtio-mem）
  • 虚拟化引擎：KVM 硬件虚拟化 / QEMU 软件虚拟化（x86_64/aarch64/riscv64）
  • 引导方式：BIOS / UEFI / UEFI+安全引导
  • 显示设备：VirtIO / VGA / VMVGA / QXL / Cirrus
  • Watchdog：i6300esb / iTCO

存储配置

  • 磁盘格式：QCOW2（动态分配、快照） / RAW（高性能）
  • 驱动类型：VirtIO（推荐） / SCSI / SATA / IDE
  • 系统盘：10-2000GB，OS 建议值（Linux 25G / Windows 60G）
  • 额外磁盘：独立配置大小、格式、驱动、IOPS
  • IOPS 限制：基于令牌桶算法，支持总 IOPS 或读写分离
  • ISO 管理：多 ISO 挂载，光驱热插拔

网络配置

  • 网卡类型：VirtIO（推荐） / e1000e / rtl8139
  • VPC 网络：交换机 + DHCP + NAT 网关 + 安全组
  • 安全组：入站/出站规则，TCP/UDP/ICMP，CIDR/交换机/安全组目标
  • 流量管控：带宽限制 + 流量配额 + 端口转发
  • 网络诊断：连通性、DNS、端口、路由追踪、带宽测试

系统配置

  • 引导顺序：硬盘/光驱/网络/USB，支持拖拽排序
  • 开机自启：宿主机启动时自动启动，支持顺序控制
  • 热修改：CPU 核心数和内存支持热插拔
  • 监控指标：CPU、内存、磁盘 I/O、网络流量、运行时间

高级设置

  • 开发者选项：CPU 冻结、APIC、PAE
  • RTC 时钟：UTC/本地时间/自定义偏移，起始日期
  • QEMU Guest Agent：文件系统冻结/解冻、网络信息、密码设置
  • 动态内存后端：气球调度 vs 弹性内存
  • SMBIOS：制造商、产品名称、序列号、UUID
  • Domain XML 编辑：管理员完全编辑权限
  • 首次重启模式：正常重启 / 宿主冷启动（解决 Sysprep 黑屏）

硬件直通

  • 支持设备：GPU（消费级/专业/计算）、NVMe 硬盘、网卡
  • VFIO：将设备绑定到 vfio-pci 驱动
  • 前置条件：IOMMU 支持、独立 IOMMU 组、VT-d/AMD-Vi
  • NVIDIA 特殊处理：自动配置隐藏虚拟化参数（解决 Error 43）

快照管理

特性内部快照外部快照
存储方式保存在原磁盘文件创建 overlay 磁盘文件
恢复速度快（virsh snapshot-revert）慢（需合并 overlay）
适用场景关机状态、少量快照运行中、频繁快照
  • 配额：管理员无限制，普通用户按配置限制
  • NVRAM 兼容：自动检测 UEFI pflash 格式，支持自动修复

磁盘迁移与扩容

  • 本机磁盘迁移：热迁移（virsh blockcopy） / 冷迁移（稀疏复制+rebase）
  • 链式磁盘处理：预创建 overlay → blockcopy --pivot / 修正 backing 路径
  • 离线磁盘扩容：FnOS（ext: resize2fs） / Windows（NTFS: ntfsresize）
  • 恢复分区处理：备份→删除→扩展→重新创建→恢复→修复

3. 网络

网络概览

  • OVS 状态检测（运行正常/需要关注）
  • 宿主机网桥管理（OVS / Linux 桥接）
  • 物理网卡列表（名称、状态、MAC、IP、默认路由）
  • OVS 端口列表（端口名、ofport、类型、关联 VM、IP）
  • 一键检测与修复

VPC 交换机

配置项说明
名称唯一标识名称
目标网桥绑定的 OVS 网桥（默认 br-ovs）
VLAN ID1-4094，用于网络隔离
安全策略allow_promiscuous / allow_mac_change / allow_forged_transmits
流量配额月下行/上行流量（GB），下行/上行带宽（Mbps）

安全策略技术原理：通过 OpenFlow 规则控制杂模式（no-flood/flood）、MAC 地址过滤（priority 90/85）、防伪传输。

安全组策略

  • 方向：入站（ingress）/ 出站（egress）
  • 协议：TCP / UDP / ICMP / 全部
  • 目标类型：CIDR / 交换机 / 安全组
  • 底层：nftables 表 kvm_console_vpc_acl，ingress/egress 链
  • 规则叠加：多个安全组规则叠加生效（入站/出站各取并集）

端口转发

  • 基于 iptables DNAT/SNAT 规则
  • 支持 TCP / UDP / both
  • 自动生成完整访问地址（tcp://IP:port）
  • TCP 探测验证连通性
  • 入站区域限制（继承全局/不受限制）
  • 用户白名单 + VM 白名单 + 建站探测

ACL 管理

  • ACL 预览：查看所有安全组生成的 nftables 规则
  • ACL 应用：加载到 kvm_console_vpc_acl 表
  • 规则模板、备份与恢复、规则监控

4. 公网 IP

三种绑定模式

模式实现VM 内配置
1:1 NAT（推荐）iptables DNAT/SNAT 映射保持私网 IP
经典网络-路由路由表项直接路由需配置公网 IP
经典网络-桥接OVS 网桥二层直通需配置公网 IP

关键功能

  • 规则预览与试算
  • 迁移公网 IP（原子性绑定关系更新）
  • 防伪造机制（OVS OpenFlow 流表规则）
  • 规则持久化：/etc/kvm-console/public-ip/rules.sh
  • 配额管理（管理员无限制）
  • 所有操作通过任务队列异步执行 + 高风险二次验证

5. 防火墙

宿主机防火墙

  • 基于 UFW，管控宿主机入站端口
  • 自动保护 SSH 和面板端口（不可删除）
  • Docker 兼容（不写入 Docker 链，保持转发 allow）
  • 端口转发联动：新增端口转发自动写入 UFW 放通规则
  • 规则来源标识：kvm-console:protected / kvm-console:port-forward

KVM 网络防火墙

  • 独立 nftables 表 kvm_console_fw，仅管控 VM 转发流量
  • 全局策略：虚拟网桥、虚拟机网段、出站/入站区域限制
  • GeoIP 区域数据（IPdeny）+ 本地导入 + 在线更新
  • VM 覆盖策略：继承全局/关闭管控/仅允许入站/仅允许区域/阻断区域
  • 白名单优先级最高，默认包含内网保留地址段
  • 紧急回滚：nft delete table inet kvm_console_fw

连接管理

  • 基于 ss -Htn state established 获取 TCP 连接
  • 两种模式：非防火墙端口连接 / 全部连接
  • 先预览再确认的安全流程
  • 基于 UFW allow 规则的端口放行判断

6. 存储池

  • 管理员专属（路由 /storage-pool/list）
  • 概览统计：总容量、已用空间、可用空间、存储池数量
  • 磁盘分组卡片：物理硬盘信息（device_path、type、model、size）
  • 状态标签：默认、已启用、LVM、可用于虚拟机
  • 操作：配置、设为默认、格式化挂载（危险操作）

7. 节点管理

  • 管理员专属（路由 /nodes）
  • 节点信息：名称、面板地址、SSH 信息、状态（在线/异常/未知）
  • 连通性探测：9 项环境检查（virsh、qemu-img、rsync、ssh、OVS 等），120s 超时
  • 认证：API ID + API Key（AES-GCM 加密存储），SSH 用户名+密码
  • 热迁移原理：预检→内存传输→脏页同步→最终切换→恢复运行
  • 冷迁移原理：停止→复制磁盘→传输配置→更新定义→清理源节点

8. 我的存储

  • 用户个人存储（路由 /my-storage，轻量云用户除外）
  • 存储类别：ISO 镜像、文件共享、虚拟磁盘
  • 挂载协议：9p VirtFS（mount -t 9p -o trans=virtio,version=9p2000.L）
  • 支持的磁盘格式：QCOW2、RAW、VMDK、VHD、VHDX、IMG
  • 访问模式：只读（RO） / 读写（RW）
  • 配额监控：绿（0-70%）/ 黄（70-90%）/ 红（90-100%）/ 超限只读
